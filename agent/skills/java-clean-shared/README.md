# java-clean-shared

Shared reference material for the local `java-clean-*` skills.

This folder is not a standalone skill. It holds common audit guidance, severity definitions, labels, finding format, and fixture maintenance notes used by:

- `java-clean-audit`
- `java-clean-code`
- `java-clean-solid`

## When to open this folder

Use it when you want to understand or maintain the shared conventions behind Java clean-code and SOLID reviews, especially:

- how findings are ranked and labeled
- what evidence is expected for a finding
- how broad audits should be approached
- how example fixtures are checked

## Related files

- `references/taxonomy.md` - severity, evidence strength, labels, and finding schema
- `references/audit-playbook.md` - shared audit workflow and review conventions
- `examples/RUNBOOK.md` - fixture maintenance and manual evaluation loop
